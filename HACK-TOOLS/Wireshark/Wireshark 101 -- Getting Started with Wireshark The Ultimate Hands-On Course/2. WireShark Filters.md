## 2. Top Wireshark Display Filters you need to know : 


1. **Common Filters :** 

	**ðŸ”¹ By Protocol**
	
	- `http` â†’ Show only HTTP traffic
	- `dns` â†’ Show only DNS queries/responses
	- `icmp` â†’ Show only ICMP (ping) packets
	- `tcp` / `udp` â†’ Show only TCP or UDP
	
	**ðŸ”¹ By IP Address**
	
	- `ip.addr == 192.168.1.10` â†’ Any packet with this IP as source or destination
	- `ip.src == 192.168.1.10` â†’ Packets sent _from_ that IP
	- `ip.dst == 192.168.1.10` â†’ Packets sent _to_ that IP
	
	**ðŸ”¹ By Port**
	
	- `tcp.port == 80` â†’ Any TCP traffic on port 80 (HTTP)
	- `udp.port == 53` â†’ Any UDP traffic on port 53 (DNS)
	- `tcp.dstport == 443` â†’ Packets going to HTTPS
	
	**ðŸ”¹ By Keywords**
	
	- `http.request` â†’ Show only HTTP requests
	- `dns.qry.name == "example.com"` â†’ DNS query for a specific domain

2.  **Operators used in filters**

	 -  **Basic Comparison Operators**

| Operator | Meaning          | Example                  |
| -------- | ---------------- | ------------------------ |
| `==`     | Equal to         | `ip.src == 192.168.1.10` |
| `!=`     | Not equal to     | `tcp.port != 80`         |
| `>`      | Greater than     | `frame.len > 500`        |
| `<`      | Less than        | `tcp.port < 1024`        |
| `>=`     | Greater or equal | `frame.len >= 1500`      |
| `<=`     | Less or equal    | `frame.len <= 64`        |

---

-  **Logical Operators**

| Operator | Meaning                 | Example                      |
| -------- | ----------------------- | ---------------------------- |
| `&&`     | AND (both must be true) | `ip.src == 10.0.0.5 && tcp`  |
| `!`      | NOT (negate)            | `!arp` (exclude ARP packets) |

---

- **Special / Pattern-Matching Operators**

|Operator|Meaning|Example|
|---|---|---|
|`contains`|Field contains a substring|`http.host contains "example"`|
|`matches`|Field matches a regex|`http.host matches ".*\.evil\.com"`|
|`in`|Value is in a set|`tcp.port in {80 443 8080}`|

---

**Tip:**

- `contains` is _faster_ than regex (`matches`)â€”use regex only when necessary.
- `in` is _gold_ for catching multiple ports, IPs, or protocols in one clean filter.
- Wrap complex logic in parentheses so you donâ€™t get tripped up by operator precedence.


3.  **Some Combo's :**

	**ðŸ”¹ Host & Protocol Combos**

	**ðŸ”¹ Credential Leaks / Cleartext**
	
	- `tcp.port == 21 || tcp.port == 23 || tcp.port == 110` â†’ FTP, Telnet, POP3 (common plaintext creds)
	- `frame contains "password"` â†’ Packets with the word â€œpasswordâ€ in payload
	
	**ðŸ”¹ Malware / Beacon Detection**
	
	- `dns.qry.name contains "xyz123"` â†’ Suspicious domain lookups
	- `tcp.flags == 0x02` â†’ SYN packets (possible scanning)

	**ðŸ”¹ File Transfer Detection**

	**ðŸ”¹ VoIP / Multimedia**

## 3. Filtering for IP Addresses : 

1.  Use `ip.addr == {ip_address}`

	- Drag the destination ip address into the filter box, and change the name from `ip.dst` `-->`  `ip.addr`.

2.  What if you want to listen to the conversation b/w two IP Addresses 

	- **Right click on a packet** `-->` **Conversation Filter** `-->` **IPv4**

	- `ip.addr eq [ip_1] and ip.addr eq [ip_2]`

3.  Hide a particular ip .

	-  `!ip.addr == {ip_address}`
